# 🚀 빅데이터 D-Day 앱: 우분투 서버 배포 가이드

안녕하세요! 이 문서는 중학생도 따라 할 수 있을 정도로 아주 쉽게 설명된 **빅데이터 D-Day 앱 배포 가이드**입니다.
이 과정을 따라 하면 내 컴퓨터에서 만든 웹사이트를 전 세계 어디서든 접속할 수 있게 됩니다!

---

## 📋 준비물
1.  **내가 만든 앱 파일들** (이 폴더 안에 있는 파일들)
2.  **우분투(Ubuntu) 서버** (켜져 있어야 해요)
3.  **파일 전송 프로그램** (FileZilla 같은 거나 `scp` 명령어를 사용할 줄 알면 더 좋아요)
4.  **공유기 접속 권한** (`iptime` 등 관리자 페이지)

---

## 1단계: 파일 옮기기 (Windows ➡️ Ubuntu)

우리가 만든 코드를 서버로 옮겨야 합니다.
`bigdata-dday` 폴더 전체를 우분투 서버의 원하는 곳(예: `~/bigdata-dday`)으로 복사해 주세요.

**옮겨야 할 필수 파일 목록:**
- `src` 폴더 (전체)
- `public` 폴더 (전체)
- `package.json`
- `package-lock.json`
- `vite.config.js`
- `index.html`
- `Dockerfile` (방금 만듦)
- `nginx.conf` (방금 만듦)
- `docker-compose.yml` (방금 만듦)

---

## 2단계: 우분투 서버에서 Docker 설치하기

우분투 서버에 접속해서 터미널(검은 창)을 열고 아래 명령어들을 복사+붙여넣기 하세요.
이 명령어들은 앱을 실행할 **Docker(도커)** 라는 프로그램을 설치해 줍니다.

```bash
# 1. 패키지 목록 업데이트
sudo apt-get update

# 2. Docker 자동 설치 스크립트 다운로드 및 실행 (가장 쉬운 방법!)
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 3. Docker Compose 설치 (이것도 필요해요)
sudo apt-get install -y docker-compose-plugin
```

설치가 잘 됐는지 확인하려면:
```bash
sudo docker --version
```
`Docker version ...` 처럼 나오면 성공입니다!

---

## 3단계: 앱 실행하기 (마법의 명령어 ✨)

이제 앱을 실행해 봅니다. 파일을 옮겨둔 폴더로 이동해서 딱 한 줄만 입력하세요.

```bash
# 1. 프로젝트 폴더로 이동 (예시)
cd ~/bigdata-dday

# 2. 실행 명령어 (백그라운드에서 실행)
sudo docker compose up -d --build
```

- 처음에 뭔가 다운로드하고 설치되는 글자들이 주르륵 뜰 거예요. (몇 분 걸릴 수 있어요)
- 마지막에 `Started` 라고 뜨면 성공입니다!

---

## 4단계: 외부 접속 설정 (공유기 포트포워딩)

서버에서는 앱이 켜졌지만, 밖에서(`goob.iptime.org`) 들어오려면 **공유기 문을 열어줘야 합니다.**

1.  **iptime 관리자 페이지** 접속 (보통 `192.168.0.1`)
2.  **[고급 설정]** -> **[NAT/라우터 관리]** -> **[포트포워드 설정]** 메뉴로 이동
3.  **[새 규칙 추가]**:
    - **내부 IP 주소**: 우분투 서버의 IP 주소 (터미널에서 `ip addr` 쳐서 확인, 보통 `192.168.0.xxx`)
    - **외부 포트**: `80` (또는 원하는 포트)
    - **내부 포트**: `80`
    - **프로토콜**: TCP
    - **적용** 클릭!

### 💡 꿀팁: 앱을 끄거나 다시 켜고 싶다면?

- **끄기**: `sudo docker compose down`
- **수정 후 재배포**: 코드를 고치고 서버에 다시 올린 뒤, `sudo docker compose up -d --build` (다시 실행하면 알아서 업데이트됩니다)
